# Running

Running is a static website for showcasing personal workout tracks. It is built with React, Vite, Tailwind CSS, and Mapbox, and displays synced workout data as map routes, yearly filters, monthly mileage charts, activity tables, a GitHub-style heatmap, and a grid route view.

- [yihong0618/running_page](https://github.com/yihong0618/running_page)

## Features

- Top summary: total distance, activity count, total duration, average pace, average heart rate, longest single activity, and PB.
- Year filter: view tracks by year, or switch to Total to see all data.
- Map view: displays workout routes and focuses on an activity when a table row or heatmap date is selected.
- Monthly mileage: bar chart for monthly distance in the selected year.
- Activity table: sortable by type, distance, elevation gain, pace, heart rate, duration, and start time.
- Total view: displays `assets/github.svg` and `assets/grid.svg`.

## Tech Stack

- Frontend: React 18, Vite, TypeScript, Tailwind CSS
- Map: Mapbox GL, react-map-gl
- Charts and SVG: Recharts, custom Python SVG generation scripts
- Data sync: Python scripts under `run_page`
- Deployment: Vercel or GitHub Pages

## Local Development

Requirements: Node.js 20+, Python 3, and pnpm.

```bash
pip3 install -r requirements.txt
corepack enable
pnpm install
pnpm dev
```

Build the static site:

```bash
pnpm build
```

The build output is generated in `dist`.

## Data Sync

The site reads data from `src/static/activities.json`. This file is usually generated by the sync scripts under `run_page`.

### Garmin

Copy the example config file and fill in your account information:

```bash
cp config-example.yaml config.yaml
python3 run_page/garmin_sync.py
```

### Strava

1. Create a Strava API application and get your `client_id` and `client_secret`.
2. Open the authorization URL and retrieve the `code`.

```text
https://www.strava.com/oauth/authorize?client_id=YOUR_CLIENT_ID&response_type=code&redirect_uri=http://localhost/exchange_token&approval_prompt=force&scope=read_all,profile:read_all,activity:read_all
```

3. Exchange the `code` for a `refresh_token`.

```bash
curl -X POST https://www.strava.com/oauth/token \
  -F client_id=YOUR_CLIENT_ID \
  -F client_secret=YOUR_CLIENT_SECRET \
  -F code=YOUR_CODE \
  -F grant_type=authorization_code
```

4. Sync your data.

```bash
python3 run_page/strava_sync.py YOUR_CLIENT_ID YOUR_CLIENT_SECRET YOUR_REFRESH_TOKEN
```

### Keep

```bash
python3 run_page/keep_sync.py YOUR_MOBILE_NUMBER YOUR_PASSWORD
```

## Generate SVGs

The Total page loads `assets/github.svg` and `assets/grid.svg`. Small yearly images are generated as circular SVGs in `assets/year_*.svg`.

```bash
python3 run_page/gen_svg.py --from-db --title "Calendar" --type github --athlete "HY Kong" --output assets/github.svg --use-localtime --min-distance 0.5
python3 run_page/gen_svg.py --from-db --title "Data Over 5km" --type grid --athlete "HY Kong" --output assets/grid.svg --min-distance 4.7 --use-localtime
python3 run_page/gen_svg.py --from-db --type circular --use-localtime
```

## Automatic Updates

The GitHub Actions workflow is located at `.github/workflows/run_data_sync.yml`.

Common environment variables:

- `RUN_TYPE`: data source, such as `strava`, `garmin`, or `keep`.
- `ATHLETE`: name shown in generated images.
- `TITLE`: title for the GitHub-style heatmap.
- `MIN_GRID_DISTANCE`: minimum distance filter for the grid image.
- `TITLE_GRID`: title for the grid image.
- `BUILD_GH_PAGES`: whether to publish GitHub Pages after syncing.

Common secrets:

- `STRAVA_CLIENT_ID`
- `STRAVA_CLIENT_SECRET`
- `STRAVA_CLIENT_REFRESH_TOKEN`
- `KEEP_MOBILE`
- `KEEP_PASSWORD`
- `GARMIN_SECRET_STRING`
- `GARMIN_SECRET_STRING_CN`

GitHub Pages setup:

1. Go to Settings -> Actions -> General -> Workflow permissions, then select Read and write permissions.
2. Go to Settings -> Pages -> Build and deployment -> Source, then select GitHub Actions.
3. Go to Actions -> Run Data Sync -> Run workflow, then manually trigger a sync and deployment.

## Refresh Existing Data

To regenerate all data, clean the old generated files first and then run the sync process again.

```bash
pnpm run clean
```

On Windows, WSL or Git Bash is recommended because the clean script uses Unix-style commands.

## Styling Entry Points

- Page title and base metadata: `index.html`, `src/static/site-metadata.ts`
- Overall page layout: `src/components/Layout/index.tsx`
- Global background, scrollbars, and Mapbox controls: `src/styles/index.css`
- Tailwind theme colors and border radius: `tailwind.config.js`
- Header navigation: `src/components/Header/index.tsx`
- Top statistics: `src/components/DashboardStats`
- Year filter and map appearance: `src/components/RunMap`
- Table styling: `src/components/RunTable/style.module.css`
- Total SVG container: `src/components/SVGStat`
- Workout types and route colors: `src/utils/const.ts`

## Configuration Tips

- Map token: update `MAPBOX_TOKEN` in `src/utils/const.ts`.
- Chinese mode: update `IS_CHINESE` in `src/utils/const.ts`.
- Map labels: update `ROAD_LABEL_DISPLAY` in `src/utils/const.ts`.
- Privacy mode: update `PRIVACY_MODE` in `src/utils/const.ts`.
- Default map height: update `MAP_HEIGHT` in `src/utils/const.ts`.

## Fork Checklist

- Remove sample data and sync again: `src/static/activities.json`, `run_page/data.db`.
- Configure the required secrets for your data source.
